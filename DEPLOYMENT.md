# Deployment guide (runbook)

The app is two deployables plus a database. Everything is verified locally; this
is the runbook for going live.

```
Next.js frontend (Vercel)  ──HTTPS/JWT──►  FastAPI backend (Render)  ──►  Postgres (Render)
```

## 1. Database — already provisioned

`lumina-creators-db` on Render (Oregon, PostgreSQL 18). Grab its **External
Database URL** from the Render dashboard for the backend's `DATABASE_URL`.

> The free instance expires ~1 year after creation — upgrade to a paid plan
> before relying on it for real client data.

## 2. Backend → Render

1. New → **Blueprint**, point at this repo, root `backend/` (uses `render.yaml`).
2. Set the `sync:false` env vars in the dashboard:
   - `DATABASE_URL` — the external URL from step 1
   - `CORS_ORIGINS` — your Vercel URL (e.g. `https://lumina-creators.vercel.app`)
   - `API_PUBLIC_URL` — this service's own URL (e.g. `https://lumina-creators-api.onrender.com`)
   - `R2_*` — Cloudflare R2 credentials for uploads (**required in prod** — the
     local-disk fallback is disabled when `ENVIRONMENT=production`)
   - `JWT_SECRET` is auto-generated by Render; `ENVIRONMENT=production` is preset.
3. The build runs `alembic upgrade head` automatically.
4. Startup **hard-fails** if `JWT_SECRET` is weak, `DATABASE_URL` is missing, or
   R2 is unconfigured — this is intentional (see `app/core/config.py`).

## 3. Frontend → Vercel  *(hold until API key provided)*

1. New Project → import this repo → **Root Directory: `frontend/`**.
2. Env var: `NEXT_PUBLIC_API_URL` = the Render backend URL from step 2.
3. Deploy. `vercel.json` pins the Next.js framework + build command.
4. Add the resulting Vercel URL to the backend's `CORS_ORIGINS` and redeploy the
   backend.

## 4. Seed an admin

Admins/clients have no public signup. After the DB is live, run once from a
machine with the prod `DATABASE_URL` (or a one-off Render shell):

```bash
ENVIRONMENT=development python scripts/seed_dev.py   # dev creds — change immediately
```

For production, create the first admin with a strong password via a secure path
rather than the dev seed. Creators self-serve at `/signup`.

## Pre-flight checklist

- [ ] Backend `CORS_ORIGINS` contains the exact Vercel origin (no trailing slash)
- [ ] `NEXT_PUBLIC_API_URL` points at the Render backend (https)
- [ ] R2 bucket + credentials set (uploads fail closed without them in prod)
- [ ] `alembic upgrade head` ran against the prod DB (Render build does this)
- [ ] First admin account created with a strong password
